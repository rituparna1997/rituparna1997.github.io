Docker Introduction

Published On 22nd Jun 20245 min read0 comments

Docker is a containarization platform. Docker used to package application code + Application dependancies for easy execution.

DOCKER USES

• Using docker we will create Docker images,

• Docker image containes code + dependancies.

• We can run docker images in any machine . It will take care of dependencies.

• When we run docker images it will create DOCKER CONTAINER,

• Docker container will contain our application.

• Docker is used to simplify APPLICATION DEPLOYMENT Process in multiple environment.

• EX---->> DEV, PRODUCTION ENV, QA ENV, UAT ENV...

IF WE ACCESS THE CONTAINER THEN WE CAN ABLE TO ACEES THE APPLICATION.

what is Application & Dependancy?
  • Application is set of code /set of Pogrames.
  • Dependances is necessary software & configuration to run that Application smoothly.
CONTAINER :
  • IF WE RUN THE DOCKER IMAGE IT WILL CREATE THE DOCKER CONTAINER.
VIRTUALIZATION VS CONTAINERIZATION
  • Virtualization means using multiple operating system in same machine.
  • Hypervisor software will be used to create virtualization.
  • It is old technique to run multiple application on same machine.
  • System performance will become very slow with virtualization.
  • To overcome the virtualization problem we are going for containerzation.
  • Virtualization is old technique
CONTAINERIZATION:
  • Containerization is new technique.
  • Container is used to package all the softwares & application code in one container for execution.
  • Container will take care of everything which is required to run our application. We can run the containers in multiple machines easily.
  • Using Dockerfile we can create container for our application.
